# Web3 行情监控与半自动下单工具用户使用手册

## 1. 项目用途

本工具用于本地查看交易所公共行情，并按用户配置的价格条件触发提醒。第一版默认使用模拟下单，用户在前端确认后，系统会生成一条模拟订单记录。

当前版本适合做这些事情：

- 查看本地规则的行情监控状态
- 配置目标价格触发条件
- 接收触发提醒
- 手动确认模拟下单
- 查看触发记录和订单记录

当前版本不适合直接用于真实资金交易。真实下单能力已经预留配置项，但第一版没有开放完整真实交易流程。

## 2. 启动前准备

需要先确认本机已经安装：

- Node.js
- pnpm

可以在项目根目录执行：

```bash
node --version
pnpm --version
```

如果能看到版本号，说明基础环境可用。

## 3. 安装依赖

进入项目根目录：

```bash
cd d:\demo\web3
```

安装依赖：

```bash
pnpm install
```

如果安装时提示 `better-sqlite3` 构建脚本被忽略，并且后端启动时报 SQLite 绑定文件缺失，可以执行：

```bash
pnpm rebuild better-sqlite3 esbuild @parcel/watcher
```

## 4. 启动项目

在项目根目录执行：

- 启动前后端服务：
  ```bash
  pnpm dev
  ```

- 启动后端，默认 http://localhost:3001
  ```bash
  pnpm dev:server
  ```

- 启动前端，默认 http://localhost:5173
  ```bash
  pnpm dev:web
  ```

启动成功后访问：

```txt
http://localhost:5173
```

后端健康检查地址：

```txt
http://localhost:3001/api/health
```

如果健康检查返回 `status: ok`，说明后端服务正常。

## 5. 配置说明

项目提供 `.env.example` 作为配置模板。需要自定义配置时，可以复制一份：

```bash
copy .env.example .env
```

常用配置：

| 配置项 | 说明 | 默认值 |
| --- | --- | --- |
| `PORT` | 后端服务端口 | `3001` |
| `VITE_WS_BASE_URL` | 前端行情推送地址 | `ws://localhost:3001/ws` |
| `DATABASE_PATH` | SQLite 数据库文件路径 | `./data/web3-trading-tool.db` |
| `DEFAULT_EXCHANGE` | 默认交易所 | `okx` |
| `ENABLE_REAL_TRADING` | 真实下单总开关 | `false` |
| `EXCHANGE_HTTP_PROXY` | 交易所 HTTP 代理，本机无法直连交易所时填写 | 空 |
| `OKX_API_KEY` | OKX API Key，真实交易预留 | 空 |
| `OKX_API_SECRET` | OKX API Secret，真实交易预留 | 空 |
| `OKX_API_PASSPHRASE` | OKX Passphrase，真实交易预留 | 空 |
| `BINANCE_API_KEY` | Binance API Key，真实交易预留 | 空 |
| `BINANCE_API_SECRET` | Binance API Secret，真实交易预留 | 空 |

公共行情和本地 WebSocket 推送不需要 API Key。API Key 只用于后续真实下单、余额查询和私有订单状态订阅。

如果本机浏览器可以访问 OKX，但 Node 后端提示无法解析或连接交易所域名，可以设置：

```txt
EXCHANGE_HTTP_PROXY=http://127.0.0.1:7897
```

这里的端口需要替换成你本机代理软件实际监听的端口。

## 6. 页面功能说明

### 监控总览

用于查看整体状态：

- 启用规则数量
- 待确认触发数量
- 订单记录数量
- 行情缓存数量
- 实时价格曲线
- 最新行情列表

如果没有规则，行情缓存通常为空。创建并启用规则后，后端才会订阅对应交易对行情。

### 监控规则

用于新增、启用、停用和删除监控规则。

规则字段说明：

| 字段 | 说明 |
| --- | --- |
| 交易所 | 当前可选择 OKX 或 Binance |
| 交易对 | 建议使用 `BTC-USDT` 格式 |
| 触发方向 | 市场价大于等于目标价，或市场价小于等于目标价 |
| 目标价格 | 触发条件使用的价格 |
| 检测频率毫秒 | 策略读取行情缓存的频率，最小 `1000` |
| 下单方向 | 买入或卖出 |
| 订单类型 | 市价或限价 |
| 计价币金额 | 例如用 `50 USDT` 买入 BTC |
| 基础币数量 | 例如卖出 `0.01 BTC` |
| 限价价格 | 限价单需要填写 |
| 最大滑点百分比 | 后续真实交易风控预留 |
| 冷却时间毫秒 | 防止短时间重复触发 |
| 最大触发次数 | 单条规则最多触发几次 |
| 下单模式 | 第一版建议保持模拟下单 |
| 创建后状态 | 创建后是否立即启用 |

### 触发确认

当规则达到目标价格后，系统会生成触发事件。

用户需要在该页面手动点击确认。确认后，系统会按规则生成模拟订单记录。

第一版确认下单不会提交真实交易所订单。

### 订单记录

用于查看已经确认生成的订单记录。

可以查看：

- 交易所
- 交易对
- 买卖方向
- 订单类型
- 数量或金额
- 订单状态
- 模拟或真实模式
- 订单号
- 创建时间

第一版订单号以 `sim-okx` 或 `sim-binance` 开头时，表示模拟订单。

## 7. 新增监控规则示例

示例目标：当 BTC-USDT 市场价大于等于 `70000` 时，提醒用户确认模拟买入。

操作步骤：

1. 进入 `监控规则`
2. 点击 `新增规则`
3. 交易所选择 `OKX`
4. 交易对填写 `BTC-USDT`
5. 触发方向选择 `市场价大于等于目标价`
6. 目标价格填写 `70000`
7. 检测频率毫秒填写 `3000`
8. 下单方向选择 `买入`
9. 订单类型选择 `市价`
10. 计价币金额填写 `50`
11. 下单模式选择 `模拟下单`
12. 创建后状态选择 `启用`
13. 点击提交

规则创建后，后端会开始订阅对应行情并按频率检测。

## 8. 触发和模拟下单流程

完整流程如下：

1. 后端订阅交易对行情
2. 策略服务读取最新价格
3. 市场价达到目标价
4. 后端生成触发事件
5. 前端收到提醒
6. 用户进入 `触发确认`
7. 用户点击确认
8. 后端生成模拟订单
9. 前端可在 `订单记录` 查看结果

## 9. API Key 使用说明

第一版不需要填写 API Key。

这些功能不需要 API Key：

- OKX 公共行情
- Binance 公共行情查询
- 本地规则管理
- 本地触发事件
- 模拟下单
- 本地订单记录

这些功能后续需要 API Key：

- 真实下单
- 查询账户余额
- 查询真实订单状态
- 订阅私有订单推送
- 撤销真实订单

填写 API Key 前，请先确认真实交易功能已经完成风控和二次确认。

## 10. 常见问题排查

### 提示“行情推送连接异常，请检查后端服务是否启动”

优先检查后端是否启动：

```txt
http://localhost:3001/api/health
```

如果不能访问，重新启动：

```bash
pnpm dev
```

如果你修改了后端端口，需要同步修改：

```txt
VITE_WS_BASE_URL=ws://localhost:你的端口/ws
```

### 监控总览没有行情

先确认是否存在启用中的规则。没有规则时，后端不会订阅行情。

可以先创建一条启用的 `BTC-USDT` 规则，然后刷新页面观察。

### 规则一直没有触发

检查这些字段：

- 目标价格是否离当前市场价太远
- 触发方向是否选反
- 规则是否启用
- 最大触发次数是否已经用完
- 冷却时间是否还没结束

### 运行状态显示“异常”

点击规则行右侧的 `错误` 查看原因。

常见原因：

- 本机 DNS 无法解析交易所域名
- Node 后端没有走系统代理
- 代理软件没有启动
- 交易所接口短暂不可用

如果错误里提示 `请检查 DNS 或 EXCHANGE_HTTP_PROXY`，可以在 `.env` 中增加：

```txt
EXCHANGE_HTTP_PROXY=http://127.0.0.1:7897
```

修改 `.env` 后需要重启 `pnpm dev`。

### 后端启动时报 SQLite 绑定错误

可以尝试：

```bash
pnpm rebuild better-sqlite3 esbuild @parcel/watcher
```

如果仍然失败，检查 Node.js 版本和本机编译环境。

### API Key 填了还是不能真实下单

第一版没有开放真实下单流程。即使填写 API Key，默认仍然是模拟下单。

## 11. 第一版限制

当前版本存在这些限制：

- OKX 支持公共行情订阅和模拟下单
- Binance 只预留适配器和公共行情查询
- 没有开放真实下单签名
- 没有账户余额校验
- 没有交易规则精度校验
- 没有私有订单状态推送
- 规则支持新增、启停、删除，暂不支持编辑

## 12. 安全提醒

- 第一版建议只使用模拟下单
- 不要把 API Key 写进前端代码
- 不要把 `.env` 提交到 Git
- 真实交易前必须补齐余额校验、价格二次确认、交易规则校验和订单状态追踪
- 如果开启真实交易，建议使用权限最小化的 API Key
- 不建议给 API Key 开启提现权限
