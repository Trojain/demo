# Web3 行情监控工具进度

## v0.1.0  2026-05-26

### 已完成

- 初始化 `pnpm workspace`，拆分为 `apps/web` 和 `apps/server`。
- 前端使用 `Vite + React + TypeScript + Ant Design 5 + Pro Components + Zustand + axios + echarts-for-react + Sass`。
- 后端使用 `Fastify + TypeScript + SQLite + ws + zod + decimal.js`。
- 建立统一交易所适配器接口，当前 OKX 支持公共行情订阅和模拟下单，Binance 预留行情查询和模拟下单。
- 建立 SQLite 三张核心表：`monitor_rules`、`trigger_events`、`order_records`。
- 实现监控规则新增、启停、删除、列表。
- 实现策略扫描，按规则检测市场价与目标价。
- 实现触发事件推送，前端通过 WebSocket 接收。
- 实现触发确认后模拟下单，并写入订单记录。
- 增加 `.env.example` 和 `README.md` 说明配置项与启动方式。

### 已确认决策

- 第一版使用 SQLite，后续多人或服务端部署时再迁移 PostgreSQL。
- 第一版默认模拟下单，真实下单需要显式开启总开关并补齐签名和风控。
- 行情数据只放内存，规则、触发事件和订单记录落库。
- 业务层通过 `ExchangeAdapter` 兼容 OKX 和 Binance，避免交易所字段进入策略层。

### 待办

- 补充 OKX 真实下单签名、余额校验和订单状态查询。
- 补充 Binance WebSocket 行情订阅和真实下单签名。
- 增加交易规则校验，例如最小数量、价格精度和最小成交额。
- 增加私有订单推送，用于跟踪部分成交、完全成交和撤单。
- 增加规则编辑能力。
- 增加审计日志导出能力。

### 已知坑位

- 市价买入在不同交易所对 `baseQuantity` 和 `quoteAmount` 的支持不同，后续真实下单前必须在 Adapter 内单独处理。
- WebSocket 行情断线重连已经预留，但 Binance 第一版未接入持续订阅。
- 当前前端确认下单时未做二次价格弹窗展示，后续真实交易前需要补齐。

- 持仓收益页首屏加载和手动刷新改为合并模式，接口旧值不会覆盖已收到的实时行情。
- 持仓收益页估值时间改为只在接受到更新行情时推进，避免显示时间前进但价格未更新。
- 交易计划 Drawer 增加行情源择优逻辑，实时 ticker 只有在时间戳更新时才覆盖表格快照。
- 最新行情表首屏加载后立即和当前实时 ticker 缓存合并，避免等待下一次推送才修正价格。
- 持仓估值接口新增 `marketEventTime` 字段，前后端可以统一判断估值价格的新旧。

### 已确认决策

- 行情新旧判断统一使用 `eventTime`，来源优先级只在时间相同或时间缺失时参与决策。
- 前端实时主源继续使用 WebSocket，REST 快照和估值接口只做补数与兜底。
- 手动刷新只能补齐缺失数据或推进更近的数据，不能把旧值重新写回页面。
- 后续新增行情消费页面时，必须复用统一行情合并工具，不能再各自维护字符串价格缓存。
