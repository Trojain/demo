# 模拟真实交易流程

本文档记录真实交易前的纸面交易设计。目标是在不触碰真实资金的前提下，先验证账户、订单、成交、持仓、收益和日志链路，并让这些记录后续可以平滑对接真实交易。

## 1. 核心目标

- 支持模拟买入、模拟卖出、持仓查看、收益统计和模拟操作日志。
- 统一模拟交易和未来真实交易的数据结构。
- 当前只使用本地 SQLite 记录模拟账户账本，不调用交易所私有接口。
- 后续真实交易接入时，尽量复用账户、持仓、成交和日志模型。

## 2. 统一账本模型

当前账本使用这些表：

| 表名 | 说明 |
| --- | --- |
| `trade_accounts` | 交易账户，当前记录模拟账户，后续可记录真实账户映射 |
| `trade_positions` | 持仓记录，按账户和交易对维护数量、成本和已实现盈亏 |
| `trade_fills` | 成交记录，记录每次买入或卖出的成交价、数量、金额、手续费和盈亏 |
| `trade_operation_logs` | 操作日志，记录账户初始化、买入、卖出、拒绝和收益重算等动作 |

关键字段 `account_type`：

```text
simulation：本地模拟账户
real：后续真实交易账户
```

接口查询参数使用 `mode` 表示页面上的下单模式：

```text
simulation：模拟下单
real：真实下单
```

统一查询接口：

```text
GET /api/trade/accounts?mode=simulation
GET /api/trade/positions?mode=simulation&exchange=okx
GET /api/trade/fills?mode=simulation&exchange=okx&limit=100
GET /api/trade/logs?mode=simulation&exchange=okx&limit=100
```

后续持仓页面、成交页面和日志页面共用这组接口，通过“下单模式”列区分模拟下单和真实下单。交易执行入口仍然在监控规则和触发确认链路内，不提供独立手动交易入口。

## 3. 模拟买入流程

```text
用户在监控规则配置买入计划
-> 后台策略扫描命中
-> 生成交易信号
-> 风控检查通过
-> 生成触发确认
-> 用户在触发确认页查看下单前预览
-> 后端读取最新行情、交易规则和模拟账户余额
-> 后端返回预计成交价、数量、金额、手续费、成交后余额和账户预检明细
-> 用户人工确认
-> 后端重新执行最终校验
-> 扣减计价币余额
-> 增加基础币持仓
-> 更新平均成本
-> 写订单记录
-> 写成交记录
-> 写模拟操作日志
```

## 4. 模拟卖出流程

```text
用户在监控规则配置卖出计划
-> 后台策略扫描命中
-> 生成交易信号
-> 风控检查通过
-> 生成触发确认
-> 用户在触发确认页查看下单前预览
-> 后端读取最新行情并检查可卖出数量
-> 后端返回预计卖出金额、手续费、预计盈亏和账户预检明细
-> 用户人工确认
-> 后端重新执行最终校验
-> 扣减基础币持仓
-> 增加计价币余额
-> 计算已实现盈亏
-> 写订单记录
-> 写成交记录
-> 写模拟操作日志
```

## 5. 收益计算口径

浮动盈亏：

```text
浮动盈亏 = 当前持仓数量 * 最新市场价 - 当前剩余持仓成本
```

浮动收益率：

```text
浮动收益率 = 浮动盈亏 / 当前剩余持仓成本
```

卖出已实现盈亏：

```text
本次卖出成本 = 平均成本价 * 卖出数量
本次卖出收入 = 成交价 * 卖出数量 - 手续费
本次已实现盈亏 = 本次卖出收入 - 本次卖出成本
```

账户总权益：

```text
账户总权益 = 可用计价币余额 + 冻结计价币余额 + 持仓按最新价估值
```

账户总收益：

```text
账户总收益 = 账户总权益 - 初始本金
```

总资产历史曲线：

```text
每日总资产快照 = 当前可用余额 + 冻结余额 + 当前持仓市值
近 30 天曲线 = 每日快照按日期排列，缺失日期按前一快照延续
```
